For two decades a CMS meant a database and an editor. The editor was where the value
lived: the WYSIWYG surface, the media library, the roles and permissions, the workflow.

Agents change that. When an agent drafts the article, the editor UI stops being the
product. What matters is the **contract** the agent writes against.

## The schema is the product

A strict, validated schema is more important when a machine is writing, not less. It is
the interface between the agent and the publication.

<Callout>In Wisp, a post that fails schema validation cannot be committed — CI is the editor's red pen.</Callout>

## Git replaces the database

Every article is a commit. History, review, rollback, and provenance come for free, and
the repo is legible to the same agents that wrote it.

## Optimizing for machines that read

The last shift is the reader. Half of them are now models. So the structure an agent
produces — a summary, an FAQ, cited sources — is exactly what AI search engines extract.
